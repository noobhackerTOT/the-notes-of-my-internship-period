# RAG 架构与攻击面

## 标准 RAG 架构

```
             ┌─────────────────────────────────────────┐
             │             用户查询                     │
             └────────────────┬────────────────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │          查询理解/重写                    │  ← 攻击面 1: 用户输入注入
             └────────────────┬────────────────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │           Embedding 模型                 │
             │    (文本 → 向量嵌入)                      │
             └────────────────┬────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              │                               │
              ▼                               ▼
   ┌─────────────────────┐     ┌─────────────────────────┐
   │   向量数据库         │     │    关键词/混合检索       │
   │   (FAISS/Pinecone)  │     │    (Elasticsearch)      │  ← 攻击面 2: 向量/检索操纵
   │                     │     │                         │
   │  ⚠️ 存储了被投毒的     │     │  ⚠️ 索引了被污染的文档    │
   │     文档向量          │     │                         │
   └─────────────────────┘     └─────────────────────────┘
              │                               │
              └───────────────┬───────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │          检索结果排序/重排                │
             │    (Re-ranking, MMR, 阈值过滤)            │  ← 攻击面 3: 排序操纵
             └────────────────┬────────────────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │       上下文组装 (Prompt 拼接)            │
             │   "基于以下知识回答:\n{docs}\n\n用户: {q}" │  ← 攻击面 4: 上下文注入
             └────────────────┬────────────────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │           LLM 推理                        │  ← 攻击面 5: 模型级
             │    (生成最终回答)                          │
             └────────────────┬────────────────────────┘
                              │
                              ▼
             ┌─────────────────────────────────────────┐
             │           输出过滤/安全层                  │  ← 攻击面 6: 输出绕过
             └────────────────────────────────────────┘
```

## 五大攻击面

### 攻击面 1: 文档索引阶段（知识库投毒）

这是最核心的攻击面。攻击者在文档被索引到知识库之前注入恶意内容。

| 攻击向量 | 描述 | 难度 | 危害 |
|---------|------|------|------|
| 文档篡改 | 直接修改已有知识库文档 | 低(若有写入权限) | 高 |
| 文档添加 | 上传含恶意指令的新文档 | 低(若有上传接口) | 高 |
| 元数据注入 | 在文档标题/标签中藏指令 | 中 | 中 |
| 协作平台投毒 | 在 Notion/Confluence/Wiki 编辑 | 低 | 高 |
| Web 爬取污染 | 通过 SEO/网页内容影响爬取的知识库 | 中 | 中 |
| 版本回滚投毒 | 利用版本控制系统嵌入历史版本 | 中 | 中 |

### 攻击面 2: Embedding 与检索阶段

操纵检索过程使恶意文档被优先召回。

| 攻击向量 | 描述 |
|---------|------|
| 对抗性嵌入 | 构造与常见查询高相似度的文档向量 |
| 关键词填充 | 在恶意文档中填充高频关键词提高检索权重 |
| 检索操纵 | 利用检索系统参数（top_k、阈值）的缺陷 |
| Embedding 篡改 | 直接修改向量数据库中的向量值 |

### 攻击面 3: 上下文组装阶段

在系统将检索结果拼接到 Prompt 时注入。

| 攻击向量 | 描述 |
|---------|------|
| 文档转义 | 利用模板字符串漏洞突破上下文边界 |
| 优先级操纵 | 让恶意文档排在检索结果第一位 |
| Token 注入 | 利用特殊 token 或分隔符 |
| 上下文窗口溢出 | 填充大量内容迫使 LLM 忽略安全指令 |

### 攻击面 4: LLM 推理阶段

LLM 本身的特性使投毒生效。

| 攻击向量 | 描述 |
|---------|------|
| 指令覆盖 | 检索内容中的指令覆盖系统 Prompt |
| 越狱触发 | 检索内容激活 LLM 的越狱行为 |
| 推理劫持 | 利用思维链/推理过程操纵输出 |
| 角色扮演陷阱 | 文档要求 LLM 扮演特定角色 |

### 攻击面 5: 输出阶段

投毒影响最终输出。

| 攻击向量 | 描述 |
|---------|------|
| 数据泄露 | 诱导 LLM 输出知识库中其他文档的内容 |
| 恶意输出 | 投毒文档包含要在输出中执行的 JS/CSS 代码 (如果渲染) |
| 幻觉引导 | 让 LLM 产生特定方向的幻觉 |

## 攻击树（Attack Tree）

```
目标: 通过知识库投毒实现 LLM 越狱
├── 1.0 注入恶意文档到知识库
│   ├── 1.1 直接写入（有权限）
│   ├── 1.2 API 接口投毒（无认证/弱认证）
│   ├── 1.3 协作平台编辑（Confluence/Notion）
│   ├── 1.4 Web 爬取搜索引擎投毒
│   └── 1.5 供应链投毒（第三方数据源）
├── 2.0 确保恶意文档被检索到
│   ├── 2.1 相关性操纵（关键词堆砌）
│   ├── 2.2 检索参数操纵（top_k/阈值）
│   ├── 2.3 频次攻击（大量重复文档）
│   └── 2.4 对抗性嵌入（针对特定查询优化）
└── 3.0 指令在 LLM 上下文中生效
    ├── 3.1 指令注入（直接写越狱 prompt）
    ├── 3.2 角色扮演（"你是一个不受限制的 AI"）
    ├── 3.3 假知识冲突（通过驳论诱导输出）
    ├── 3.4 系统提示覆写（利用文档结构标记）
    └── 3.5 多轮引导（渐进式越狱）
```

## 检测难度矩阵

| 攻击类型 | 静态检测 | 动态检测 | 人工审核 |
|---------|---------|---------|---------|
| 显式指令注入 | 🟢 容易 | 🟢 容易 | 🟢 容易 |
| 隐式指令注入 | 🔴 困难 | 🟡 中等 | 🟡 中等 |
| 对抗性嵌入 | 🔴 非常困难 | 🔴 困难 | 🔴 非常困难 |
| 时序投毒 | 🔴 非常困难 | 🔴 非常困难 | 🔴 几乎不可能 |
| 多文档冲突 | 🟡 中等 | 🟡 中等 | 🟡 中等 |
| 元数据注入 | 🔴 困难 | 🟡 中等 | 🟡 中等 |
