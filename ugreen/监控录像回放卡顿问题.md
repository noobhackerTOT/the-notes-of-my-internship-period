# 监控录像回放卡顿问题

监控录像回放卡顿问题 --- 排查总结

现象

RAID6 + 4×8TB HDD，存储16路监控录像

SMB回放视频时卡顿、跳帧

顺序读取大文件正常（95 MB/s）

CPU/内存正常，网络出方向流量偏低（20-30 Mbps）

核心判断

顺序读快、随机读慢 → 问题在存储层的随机读取性能，不是网络或CPU。

---

步骤             检查项           命令/方法                   目标

1                磁盘 I/O 延迟    iostat -x 1                 看 r_await、%util 是否过高

2                RAID6 随机读     fio\--rw=randread\--bs=4k   应≥200 IOPS，低于100为异常
IOPS

3                RAID 状态        cat /proc/mdstat            确认无重建/校验进行中

4                文件碎片率       fsck.ext4 -fn /dev/md0      碎片率\>10%需整理

5                SMB 异步 I/O     检查 smb.conf 中 aio read   未启用则随机读性能差
size = 1

6                本地读对比测试   dd if=录像文件 of=/dev/null 本地慢→存储层问题；本地快→SMB问题

---

最可能的根本原因

RAID6 + 机械硬盘随机读性能天生差（需读数据块+校验块）

长时间写入导致文件碎片（监控录像循环覆盖）

SMB未启用异步I/O（单线程阻塞）

解决方案

原因 解决办法

RAID6 随机读慢 换 RAID10（牺牲容量换性能）

文件碎片 整理碎片或定期迁移数据

SMB 异步未开 smb.conf 添加 aio read size = 1 并重启

监控场景优化 改用 块级协议（iSCSI） 或 NFS（某些场景比SMB更适合视频流）

一句话结论

机械硬盘 RAID6 的随机读取能力弱 + 文件碎片 + SMB 未优化 =
监控回放卡顿。优先做本地读测试和 iostat 确认瓶颈所在。
