# 场景 1绿联 NAS 无法通过 SMB 访问，但 Ping 通

场景 1：绿联 NAS 无法通过 SMB 访问，但 Ping 通

设备：绿联 DXP4800（UGREEN NAS），系统为 UGOS Pro。

网络环境：NAS 和你的 Windows 电脑连接在同一台交换机下，IP 分别为
192.168.1.100 和 192.168.1.50。

现象：

从电脑上 ping 192.168.1.100 正常，延迟 \<1ms。

在文件资源管理器输入 \\\\192.168.1.100 后，提示"网络错误，Windows
无法访问 \\192.168.1.100"。

手机上的绿联 App 可以正常登录 NAS，能看到文件，播放视频也正常。

电脑上之前从未映射过该 NAS 的驱动器。

电脑可以正常访问互联网，也可以访问其他 Windows 电脑的共享文件夹（例如
[\192.168.1.200\share）。](\\\\192.168.1.200\\share）。)

解决方式：

1.Windows 网络配置文件为"公共" → 禁用文件和打印机共享
（"文件和打印机共享" 其实对应了一组预定义的防火墙规则，主要包括：SMB
协议（TCP 445 端口）NetBIOS 相关端口（137-139 UDP/TCP））→
解决方案：改为"专用网络"，或手动允许。

2.Windows SMB 客户端功能未启用（尤其是 SMB 1.0
被禁用后，某些旧设备或配置问题） → 解决方案：启用 SMB
2.0/3.0（默认开启），或重装 SMB 组件。

3.NAS 的 SMB 服务配置限制（如只允许 SMB 3.0，而 Windows 因某些策略只尝试
SMB 2.0） → 解决方案：检查 NAS SMB 最小/最大协议版本。

4.组策略或第三方安全软件拦截 → 解决方案：临时禁用安全软件测试。

5.Credential Manager
中存储了错误凭据（虽然场景说从未映射，但可能通过其他方式访问过） →
解决方案：清除凭据。
