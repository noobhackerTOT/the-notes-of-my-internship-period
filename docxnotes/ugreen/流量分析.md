# 流量分析

如果你已经不满足于单个协议过滤，而是想做 **SOC / 威胁狩猎（Threat Hunting）**  级别的分析，那么需要掌握一些组合过滤器。

下面这些是我在分析恶意 PCAP 时最常用的思路。

---

# 1. 找所有主动外联

这是最经典的。

```wireshark
(
    dns
    or http.request
    or tls.handshake.type == 1
    or (
        tcp.flags.syn == 1
        and tcp.flags.ack == 0
        and !(ip.dst == 10.0.0.0/8)
        and !(ip.dst == 172.16.0.0/12)
        and !(ip.dst == 192.168.0.0/16)
    )
)
and !(ssdp)
```

用途：

```text
主机访问了哪些网站
解析了哪些域名
建立了哪些HTTPS连接
连接了哪些异常端口
```

---

# 2. 找可疑下载行为

```wireshark
http.request
and (
    http.request.uri contains ".exe"
    or http.request.uri contains ".dll"
    or http.request.uri contains ".zip"
    or http.request.uri contains ".js"
    or http.request.uri contains ".ps1"
)
```

典型发现：

```text
Emotet
AgentTesla
AsyncRAT
```

下载器阶段。

---

# 3. Office → PowerShell 攻击链

虽然 Wireshark看不到进程链，但能看出特征。

```wireshark
http.request
and (
    http.user_agent contains "PowerShell"
    or http.user_agent contains "BITS"
)
```

或者：

```wireshark
frame contains "powershell"
```

发现：

```text
Invoke-WebRequest
DownloadString
IEX
```

---

# 4. 找 Beacon（C2心跳）

很多木马会固定时间回连。

```wireshark
tcp.len > 0
and !(dns)
and !(http)
and !(tls)
```

然后：

```text
Statistics
→ Conversations
```

观察：

```text
每60秒
每300秒
固定流量大小
```

例如：

```text
256B
256B
256B
256B
```

这是典型 Beacon。

---

# 5. Kerberoasting 狩猎

```wireshark
kerberos.msg_type == 12
```

即：

```text
TGS-REQ
```

再加：

```wireshark
kerberos.SNameString
```

查看：

```text
MSSQLSvc
HTTP
CIFS
LDAP
```

如果短时间大量出现：

```text
TGS-REQ
TGS-REQ
TGS-REQ
```

可能在：

```text
SPN枚举
Kerberoasting
```

---

# 6. SMB横向移动

```wireshark
smb2
and (
    smb2.cmd == 1
    or smb2.cmd == 3
    or smb2.cmd == 5
)
```

对应：

```text
Session Setup
Tree Connect
Create
```

常见攻击链：

```text
4624 Type3
↓
SMB Session Setup
↓
ADMIN$
↓
Create Service
```

---

# 7. PsExec检测

非常经典。

```wireshark
smb2
and (
    frame contains "PSEXESVC"
)
```

很多内网横向移动直接命中。

---

# 8. RDP入侵

```wireshark
tcp.port == 3389
and tcp.flags.syn == 1
and tcp.flags.ack == 0
```

统计：

```text
来源IP
连接次数
```

寻找：

```text
暴力破解
```

---

# 9. DNS隧道

高频DNS请求。

```wireshark
dns.qry.name.len > 50
```

或者：

```wireshark
dns.qry.name matches "[A-Za-z0-9]{30,}"
```

例如：

```text
ajd92jd92jd92jda.example.com
```

可能是：

```text
Iodine
Dnscat2
```

---

# 10. TLS可疑域名

```wireshark
tls.handshake.type == 1
and !(
    tls.handshake.extensions_server_name contains "google"
    or tls.handshake.extensions_server_name contains "microsoft"
    or tls.handshake.extensions_server_name contains "github"
)
```

快速排除常见域名。

---

# 11. 查所有公网通信

```wireshark
ip
and !(
    ip.addr == 10.0.0.0/8
    or ip.addr == 172.16.0.0/12
    or ip.addr == 192.168.0.0/16
)
```

经常用于：

```text
先找所有公网IP
再逐个分析
```

---

# 12. 恶意流量分析万能版

这是很多分析师打开 PCAP 后第一条过滤器。

```wireshark
(
    dns
    or http.request
    or tls.handshake.type == 1
    or kerberos
    or smb2
    or nbns
    or ldap
    or dcerpc
    or (
        tcp.flags.syn == 1
        and tcp.flags.ack == 0
    )
)
and !(ssdp)
```

直接覆盖：

```text
DNS
HTTP
HTTPS
Kerberos
SMB
NBNS
LDAP
RPC
新建连接
```

---

# 13. AD域攻击狩猎版（推荐你练）

结合你最近在学的 Kerberos、SMB、NBNS、Windows日志审计：

```wireshark
(
    kerberos
    or smb2
    or nbns
    or ldap
    or dcerpc
)
and !(
    ip.addr == DC_IP
)
```

重点观察：

```text
TGS请求异常
SMB访问异常
NBNS广播异常
LDAP枚举
RPC调用
```

---

# 14. 红队横向移动综合过滤器

```wireshark
(
    kerberos.msg_type == 12
    or smb2.cmd == 1
    or smb2.cmd == 5
    or tcp.port == 445
    or tcp.port == 135
    or tcp.port == 3389
)
```

‍

对应攻击链：

```text
Kerberos认证
↓
SMB登录
↓
文件上传
↓
服务创建
↓
RDP接管
```

这类过滤器已经接近 SOC 面试、CTF 流量分析题和 Malware-Traffic-Analysis 网站样本的实际分析套路了。很多时候不是记协议，而是围绕攻击链构建过滤器：**认证 → 横向移动 → 下载执行 → C2通信 → 持久化**。这才是高级流量分析的思维方式。
